# 影像輸出計算機

## 工具定位

這是一個靜態網頁工具，用來處理影像輸出相關的尺寸與解析度換算。

核心目標是讓使用者不用自己選擇計算模式，而是直接輸入目前已知條件，系統自動判斷可以補出哪些結果。

主要換算對象：

- 實體尺寸（mm / cm / inch）
- 像素尺寸（px）
- 解析度（DPI / PPI）
- 長寬比

---

## UX 方向

畫面分成左右兩半：

- 左側：輸入
- 右側：輸出

左右兩側欄位結構要一致，但用途不同：

- 左側代表「目前已知條件」
- 右側代表「系統依據目前條件補完後的結果」

### 核心互動規則

- 左側有填寫的值，右側直接同步顯示
- 左側沒填寫、但依目前條件可推算的值，右側自動計算後顯示
- 若條件不足，右側對應欄位保持空白
- 不提供計算模式切換，由程式自動判斷目前正在算什麼

---

## 左側輸入區需求

### 1. 預設資料快速帶入

左側最上方提供一個**可搜尋的下拉選單**，資料來源為外部 JSON 檔案。

用途：

- 讓使用者快速選擇常見預設數值組合
- 選取後直接帶入左側輸入欄位

需求：

- 下拉選單需支援搜尋
- 預設清單由 JSON 檔維護
- JSON 內容可包含多組常見輸出尺寸、裝置規格、印刷尺寸或其他預設組合
- 選定後，自動回填左側對應欄位

### 2. 尺寸輸入欄位

左側至少要有：

- 寬 + 單位（`px / mm / cm / inch`）
- 高 + 單位（`px / mm / cm / inch`）

### 3. 解析度輸入欄位

- 解析度數值輸入欄
- 單位選擇：`DPI / PPI`
- 提供常用快捷值：
  - `72`
  - `96`
  - `150`
  - `300`
  - `600`

### 4. 長寬比輸入

需支援依長寬比補算另一邊尺寸。

建議支援：

- 常用比例快速選擇
- 自訂比例輸入（例如 `16:9`）

第一版先提供兩個範例快速選項即可：

- `16:9`
- `A 系列（1:√2）`

---

## 右側輸出區需求

右側欄位結構與左側一致。

顯示規則：

- 左側已輸入的欄位，右側同步顯示
- 左側空白但可依條件推算的欄位，右側自動補出結果
- 無法推算的欄位維持空白
- 右側只顯示對應欄位結果，不額外展開所有單位換算明細
- 視覺上需區分：
  - 使用者原本已輸入的值
  - 系統自動計算補出的值
- 建議兩者使用不同顏色或不同狀態樣式呈現

---

## 必須支援的計算情境

工具要能自動根據目前輸入條件，處理以下換算：

1. 已知實體尺寸 + DPI/PPI，推算像素尺寸
2. 已知像素尺寸 + DPI/PPI，推算實體尺寸
3. 已知像素尺寸 + 實體尺寸，反推 DPI/PPI
4. 已知一邊尺寸 + 長寬比，推算另一邊尺寸

---

## 額外操作功能

### 1. 複製右側結果到左側

需提供一個按鈕，功能如下：

- 將目前右側所有結果整組複製到左側
- 複製後，左側成為新的輸入基礎
- 用於連續換算或後續微調

### 2. 清空 / 重設

需提供清空所有欄位的功能。

### 3. 交換寬高

建議提供交換寬度與高度的功能，方便橫式 / 直式切換。

## 建議檔案結構

```text
tools/image-output-calculator/
├─ README.md
├─ index.html
├─ style.css
├─ script.js
└─ presets.json
```

其中：

- `presets.json`：可搜尋下拉選單要讀取的預設資料

---

## 實作原則

- 工具必須能在瀏覽器中直接開啟使用
- 不依賴後端
- 預設資料由前端讀取 JSON
- 條件不足時，不亂算、不亂補，直接留空
- 以「自動補完」為核心，而不是要求使用者先理解公式或選擇模式

---

## 預設資料 JSON 第一版

第一版先收以下常用項目：

- A3
- A4
- 照片 4x6
- 照片 5x7

後續可再由使用者自行擴充。

---

## 視覺與狀態提示

第一版骨架需先把以下狀態區分做進去：

- 右側顯示的是「使用者輸入值」時，使用一種顏色
- 右側顯示的是「計算補完值」時，使用另一種顏色

這個狀態標示是工具辨識性的重點之一。
